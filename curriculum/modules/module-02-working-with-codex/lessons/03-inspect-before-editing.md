# Lesson 2.03: Inspect Before Editing

## Learn

Before Codex changes a file, ask it to inspect the file and explain what it recommends. This is the practical version of "explain, plan, act": understand first, then decide whether to change something.

The beginner loop is:

1. **Ask** for one clear thing.
2. **Inspect** what Codex can see.
3. **Plan** before bigger changes.
4. **Act** when you are ready for a real edit.
5. **Review** what changed.

You do not need to memorize the loop. Just remember: look first, change second.

## Practice

Try this in Codex:

```text
Inspect `practice-workspace/notes/weekly-review-draft.md` and compare it with `practice-workspace/templates/weekly-recap-template.md`. Tell me what you would improve before editing anything. Keep the recommendation short and ask me to review it before making changes.
```

**Success criteria:** Codex proposes a clear change without editing the file, and the learner understands what would happen next.
