# Lesson 2.01: Explain, Plan, and Act

## Learn

Beginners often ask Codex to do everything at once. That makes it harder to understand what happened and harder to review the result. A better pattern is to separate three modes:

- **Explain** when you want understanding
- **Plan** when you want structure before action
- **Act** when you are ready for a real change

## Practice

Try this in Codex:

```text
This course includes sample files for practice. Walk me through this in order:
1. First define explain, plan, and act in one short sentence each.
2. Then tell me, in plain English, why a beginner should not ask for all three at once.
3. Then use the sample file `practice-workspace/notes/weekly-review-draft.md` to show me one example of an explain prompt, one example of a plan prompt, and one example of an act prompt.
4. Do not edit the file yet. I only want to understand the difference.
```

**Success criteria:** Codex clearly distinguishes the three modes and uses the actual draft file as the concrete example.
