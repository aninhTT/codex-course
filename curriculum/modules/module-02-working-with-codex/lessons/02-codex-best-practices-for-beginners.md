# Lesson 2.02: Codex Best Practices for Beginners

## Learn

Codex works best when you give it a clear job, useful context, and a chance to show its thinking before it changes anything. You do not need fancy prompts. You need a few steady habits.

The beginner loop is:

1. **Ask** for one clear thing.
2. **Inspect** what Codex can see or what it found.
3. **Plan** before bigger changes.
4. **Act** only when you are ready for Codex to edit or create something.
5. **Review** what changed before you trust the result.

This loop keeps you in control. It also makes Codex feel less mysterious because you can see what it is using, what it intends to do, and what it actually changed.

The most useful beginner habits are:

- Start with the smallest useful task.
- Tell Codex whether you want explanation, planning, or action.
- Name the files or folders Codex should use when you can.
- Ask what Codex is about to change before letting it edit.
- Ask for a summary after Codex edits something.
- Stop and ask for a recap when you lose your place.

## Practice

Try this in Codex:

```text
Teach me the beginner Codex best-practice loop: Ask, Inspect, Plan, Act, Review.

Use `practice-workspace/notes/weekly-review-draft.md` as the example, but do not edit it. For each step, show me one plain-language prompt I could use and explain why that step helps me stay in control.
```

**Success criteria:** Codex explains the full loop in beginner-friendly language, uses the practice file as the example, and makes it clear that edits should come after inspection or planning, not before.
