# Lesson 3.05: Choose And Interview A Workflow

## Learn

Now the learner chooses one real workflow for a custom skill. The best beginner choice is repetitive, clear enough to describe, and small enough to test quickly.

Before Codex builds, it needs the workflow shape:

- trigger
- inputs
- steps
- desired output
- edge cases
- what to leave out of the first version

This interview step prevents a vague skill from turning into a messy skill.

## Practice

Try this in Codex:

```text
Help me choose one workflow to turn into a custom skill. Compare 2 or 3 options if I have them. Then interview me about the trigger, inputs, steps, edge cases, desired output, and what we should leave out of the first version. End with a concise workflow summary we can build from.
```

**Success criteria:** The learner ends with one clear workflow summary that is specific enough to build from.
