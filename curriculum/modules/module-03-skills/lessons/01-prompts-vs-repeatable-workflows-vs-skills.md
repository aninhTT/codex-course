# Lesson 3.01: Prompts vs Repeatable Workflows vs Skills

## Learn

A prompt is a one-time instruction. A repeatable workflow is a task you do the same way again and again. A skill is a reusable set of instructions that helps Codex perform one kind of workflow consistently.

The key beginner idea is simple: if you keep asking Codex to do the same kind of work, a skill may be the right next step.

This course uses "workflow automation" in a beginner sense here: making repeated work easier and more consistent through skills. Scheduled automations come later in Module 9.

## Practice

Try this in Codex:

```text
Explain the difference between a one-time prompt, a repeatable workflow, and a Codex skill. Use my weekly recap work from Module 2 as the example. Then tell me when it would be worth turning a repeated prompt into a skill.
```

**Success criteria:** The learner can explain why a repeated task might become a skill and understands that scheduled automations are a later topic.
