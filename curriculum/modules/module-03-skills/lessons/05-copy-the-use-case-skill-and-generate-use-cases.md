# Lesson 3.05: Copy the `use-case` Skill and Generate Use Cases

## Learn

Now that the learner has built one simple skill, it helps to study a stronger example from the outside. The `use-case` skill is useful because it shows how a skill can turn messy information into practical workflow ideas.

For this course, the real goal is not to master the entire skill. It is to copy it, inspect it, and use it to generate:

- one candidate for a future custom skill
- one candidate for a future prototype

Before starting, look at:

- `reference/use-case-skill-summary.md`
- `learner-assets/use-case-shortlist.md`

## Practice

Try this in Codex:

```text
Install the `use-case` skill from `https://github.com/aninhTT/skill-bank/tree/main/use-case` using the GitHub skill installer workflow. Then summarize how the skill works in beginner-friendly language and help me generate at least 2 or 3 use cases from my own work, including at least one that could become a custom skill and one that could become a prototype in later modules.
```

**Success criteria:** The learner installs the example skill, understands its high-level structure, and captures candidate ideas for both a custom skill and a prototype.
