# Lesson 4.01: What A Prototype Is For

## Learn

A prototype is a small working version of an idea. It helps you see and test a workflow instead of only describing it.

A skill is best when you want Codex to follow reusable instructions. A document is best when you need written output. A prototype is best when a small interface, dashboard, tracker, or tool would make the workflow easier to understand or use.

The first prototype should be small. The goal is not a perfect app. The goal is something you can inspect, preview, and improve.

## Practice

Try this in Codex:

```text
Explain what a prototype is for using one of my workflow ideas as the example. Compare when that idea should be a document, a skill, or a small app. Recommend one prototype idea that would be useful but still small.
```

**Success criteria:** The learner can explain why a prototype is different from a document or skill and has one small prototype candidate.
