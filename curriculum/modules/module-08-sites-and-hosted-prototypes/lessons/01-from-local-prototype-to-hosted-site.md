# Lesson 8.01: From Local Prototype To Hosted Site

## Learn

In Module 4, the goal was a local prototype you could inspect and improve. Sites adds a publishing path, but publishing is not the same as prototyping.

Think of the stages this way:

1. **Local preview:** useful for building and checking the experience on your machine.
2. **Saved version:** a reviewable candidate that is not automatically shared as a live production deployment.
3. **Deployment:** a production URL for a chosen audience.

The safest habit is to review locally first, then save a version when you need a candidate, and deploy only when the audience and content are ready.

## Practice

Try this in Codex:

```text
Review my Module 4 prototype and tell me whether it is a good candidate for Sites. Do not deploy anything. Explain what should be checked locally first, what would make a good saved version, and what would need to be true before deployment.
```

**Success criteria:** The learner can explain the difference between local preview, saved version, and deployment.
