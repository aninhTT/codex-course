# Lesson 8.02: Review Before You Publish

## Learn

Publishing makes review more important, not less. Before anything goes live, the learner should check the source changes, the visible page, the data handling, and the intended audience.

A Sites-readiness review should answer:

- Does the prototype work locally?
- Is the content appropriate for the intended audience?
- Are there secrets or private data in the files?
- Does the page explain or show the right workflow clearly?
- Is the deployable version the version the learner actually intends to share?

This builds directly on the "review before you trust" habit from Module 2.

## Practice

Try this in Codex:

```text
Create a Sites-readiness checklist for my prototype. Focus on source review, visible page review, data/privacy concerns, audience, and what would block publishing. Keep this as a checklist I can reuse.
```

**Success criteria:** The learner has a reusable checklist and understands that deployment should wait until review blockers are resolved.
