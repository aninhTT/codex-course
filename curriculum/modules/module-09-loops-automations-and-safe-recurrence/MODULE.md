# Module 9: Loops, Automations, And Safe Recurrence

You already practiced the manual Ask, Inspect, Plan, Act, Review loop; now you'll design safe recurring loops.

This module keeps automation beginner-safe. The learner does not need to schedule a live automation by default. The main artifact is an automation design card that makes the trigger, cadence, prompt, evidence, stop condition, risks, and review plan visible before anything runs unattended.

## What You'll Learn

- how manual loops, review loops, Goal loops, thread automations, and standalone automations differ
- how to choose the smallest recurring pattern that fits the task
- how to write a durable automation prompt
- how to test manually before scheduling
- how to define stop conditions and review early outputs
