# Lesson 9.01: The Loop Ladder

## Learn

The beginner loop from Module 2 was manual: Ask, Inspect, Plan, Act, Review. Power-user Codex work uses the same idea, but with more structure.

Think of loops as a ladder:

1. **Normal prompt loop:** you ask for the next useful step.
2. **Review loop:** Codex checks work, you review, and Codex revises.
3. **Goal loop:** Codex keeps working toward a defined finish line in the current thread.
4. **Thread automation:** Codex wakes up the same conversation on a schedule.
5. **Standalone automation:** Codex starts fresh scheduled runs and reports findings separately.

The higher rungs are not automatically better. Use the smallest loop that gives you the control you need.

## Practice

Try this in Codex:

```text
Teach me the loop ladder using one workflow from this course. Compare a normal prompt loop, review loop, Goal loop, thread automation, and standalone automation. Recommend the smallest safe option and explain why.
```

**Success criteria:** The learner can name the main loop types and explain why automation is not always the best first choice.
