# Lesson 1.03: Why Local Folders Matter

## Learn

The folder you open determines what Codex can inspect, reason about, and potentially change. Coding tools like Codex work with files and folders that live locally on your computer. That is why folder choice matters so much: the folder is not just organization, it is the working context.

Opening the whole course workspace gives Codex broad context: the modules, the practice files, the reference docs, and the starter app. Opening a smaller folder gives Codex a tighter, more focused view. You do not need to master that tradeoff yet. The main idea is simpler: Codex can only work with the files that are inside the folder you gave it.

This matters because good folder choice makes Codex both **more useful** and **safer**.

## Practice

Try this in Codex:

```text
Look at this course workspace and do three things:
1. List the main folders you can see and what each one seems to be for.
2. Explain how coding tools like Codex work with files and folders that live locally on my computer.
3. Give me one simple example of why I might open this whole course folder in Codex today, and one simple example of when I might open just a smaller project folder on another day.
```

**Success criteria:** Codex correctly identifies the main folders, explains that local coding tools operate on files and folders on the learner's computer, and gives a simple, beginner-friendly explanation of when broad versus smaller folder context is useful.
