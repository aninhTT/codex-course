# Lesson 7.02: Prepare A Clean Recording

## Learn

A good recording is short, focused, and complete. Before recording, decide the start point, the end point, the inputs that may change next time, and the proof that the workflow worked.

Keep the demonstration clean:

- use realistic but non-sensitive inputs
- close unrelated tabs or windows
- avoid secrets, private customer data, or anything you would not want captured
- stop recording when the workflow is complete
- leave unrelated cleanup for later

This is the same beginner habit from earlier modules: inspect and plan before acting. The only new twist is that the "action" is a demonstration.

## Practice

Try this in Codex:

```text
Help me prepare a clean recording plan for this workflow. Ask me for the start point, end point, variable inputs, sensitive information to avoid, and success check. Then give me a short recording script I can follow step by step.
```

**Success criteria:** The learner has a recording script that avoids secrets, names variable inputs, and defines how to verify success.
