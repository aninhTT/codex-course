# Lesson 7.03: Refine The Recorded Skill

## Learn

After a workflow is recorded, the first generated skill is a draft. Treat it the same way you treated earlier Codex edits: review before you trust.

The most useful refinements usually name things that were obvious to you but not obvious from the recording:

- which inputs change each time
- which defaults should stay the same
- which steps are preferences rather than strict rules
- how to verify the result
- when Codex should stop and ask instead of continuing

Do not turn the skill into a giant manual. Keep it focused on the workflow you demonstrated.

## Practice

After recording a workflow or drafting one manually, try this in Codex:

```text
Review this recorded workflow skill with me. Help me identify the variable inputs, hidden preferences, verification steps, and stop conditions. Then recommend the smallest edits that would make the skill safer and easier to reuse.
```

**Success criteria:** The learner reviews the skill draft and improves it without expanding it beyond the recorded workflow.
