# Lesson 7.04: Fallback If Recording Is Unavailable

## Learn

Record & Replay may not be available for every learner, device, region, or workspace policy. That is fine. The lesson still matters because the recording mindset helps you describe workflows better.

If you cannot record, use a narrated walkthrough instead. You can list the screens, clicks, decisions, inputs, expected output, and verification step. Codex can turn that into a skill plan or a draft skill just like it did in Module 4.

The fallback is not worse learning. It simply swaps "show the workflow" for "describe the workflow in recording-level detail."

## Practice

Try this in Codex:

```text
Record & Replay is not available to me right now. Help me turn my recording script into a written workflow walkthrough instead. Then use that walkthrough to draft or refine a reusable skill, keeping the same boundaries and success checks.
```

**Success criteria:** The learner can still produce a reusable workflow description or skill draft even if recording is unavailable.
