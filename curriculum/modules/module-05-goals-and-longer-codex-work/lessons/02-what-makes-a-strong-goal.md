# Lesson 5.02: What Makes A Strong Goal

## Learn

A strong Goal gives Codex enough structure to keep moving without wandering.

It should include:

- **Outcome:** what should be true when the work is done
- **Evidence:** what Codex should inspect to prove the outcome
- **Constraints:** what must stay true
- **Allowed sources:** which files, folders, tools, or sources Codex may use
- **Boundaries:** what is out of scope
- **Blocked stop condition:** when Codex should stop and explain what is missing

Weak Goal:

```text
/goal Research vendors for us.
```

Stronger Goal:

```text
/goal Create an evidence-backed vendor evaluation for the three tools named in this folder. Use only the source notes, vendor websites, and approved comparison criteria. End with a table of findings, source links, confidence levels, risks, and open questions. If a required source is missing or a claim cannot be verified, label it as blocked or uncertain instead of filling the gap.
```

## Practice

Try this in Codex:

```text
Help me evaluate this Goal idea. Check whether it has an outcome, evidence source, constraints, allowed sources, boundaries, and blocked stop condition. Then rewrite it into a stronger Goal without making it more complicated than needed.
```

**Success criteria:** The learner can identify the parts of a strong Goal and improve a weak Goal draft.
