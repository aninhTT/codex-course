# Lesson 5.01: Normal Prompt vs Goal

## Learn

A normal prompt asks Codex for the next useful step: inspect this file, make this change, summarize this folder, or help me plan.

A Goal gives Codex a persistent objective to work toward across a longer task. Use it when the path may take several steps but the finish line can still be checked.

Goal mode is not for everything. If the task is quick, vague, or impossible to verify, use a normal prompt instead.

## Practice

Try this in Codex:

```text
Teach me the difference between a normal prompt and a Codex Goal using one workflow from this course. Give me 3 examples where a normal prompt is enough and 3 examples where a Goal would be better.
```

**Success criteria:** The learner can explain when a normal prompt is enough and when a Goal is worth using.
