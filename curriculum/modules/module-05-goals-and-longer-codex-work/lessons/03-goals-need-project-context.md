# Lesson 5.03: Goals Need Project Context

## Learn

Longer Codex work depends on context. A Goal should say which project or folder Codex should use, what sources are allowed, and what should stay out of scope.

If the needed evidence lives in another project, switch projects deliberately. Summarize before leaving the course project, open the right project, and make sure the Goal names the evidence Codex can use.

The question is not only "What do I want?" It is also "Where should Codex look?"

## Practice

Try this in Codex:

```text
Take one Goal candidate from my course work and help me define its project context. Which project or folder should Codex use? What files or sources are allowed? What should stay out of scope? If I need to switch projects, give me a summary prompt before I leave and a resume prompt for when I return.
```

**Success criteria:** The learner can connect a Goal to the right project context and allowed evidence.
