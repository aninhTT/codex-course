# Lesson 5.07: Build and Iterate Your Prototype

## Learn

The purpose of this lesson is not to create a perfect app. It is to turn the approved plan into a first real version, then make one useful iteration.

Best practice: build the approved v1 first, review it, and then choose one iteration. Resist the urge to add every good idea at once.

## Practice

Step 1: type this into Codex:

```text
Now help me build the first version of this prototype in the `playground/` folder based on the plan we approved. Keep the scope tight and focused on the 2 to 3 features we agreed on. After building, tell me how to preview it and what I should review first.
```

Step 2: after the first version exists, type this:

```text
Now review this prototype with me. Tell me what is working, what feels weak, and what one useful iteration we should make next.
```

**Success criteria:** The learner builds a first version in `playground/`, reviews it with Codex, and makes at least one useful iteration instead of treating the first draft as final.
