# Lesson 5.06: Plan Your Prototype

## Learn

Before building, it helps to plan a small version first. The safest beginner move is to start with 2 to 3 features, not the full dream version.

Best practice: a good v1 plan also says what is out of scope. Leaving things out on purpose is how you keep the first build understandable.

## Practice

Try this in Codex:

```text
Let's use plan mode to plan this prototype before building it. Help me define a small v1 with only 2 to 3 features, what each feature should do, what files or folder we should build in, and what we should leave out for now. Do not build anything yet. I want to review the plan first.
```

**Success criteria:** The learner ends with a plan they understand, the scope is limited to 2 to 3 features, and the build feels realistic for a first pass.
