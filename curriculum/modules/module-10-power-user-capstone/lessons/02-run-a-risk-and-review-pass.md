# Lesson 10.02: Run A Risk And Review Pass

## Learn

Power-user workflows need review because they can touch more systems, run more often, or reach more people. A good risk pass is practical, not scary.

Ask Codex to check:

- missing evidence
- unclear ownership
- sensitive data or secret values
- automation scope that is too broad
- deployment or sharing risk
- places where Codex should ask before acting

This turns review into a normal operating habit instead of a one-time course step.

## Practice

Try this in Codex:

```text
Review my workflow system map for risks. Look for missing evidence, privacy concerns, secret values, unclear ownership, overbroad automation, deployment risk, and places where Codex should stop and ask me before acting. Then recommend the smallest fixes.
```

**Success criteria:** The learner identifies risks and fixes the smallest important issues before treating the system as ready.
