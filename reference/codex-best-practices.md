# Codex Best Practices for Beginners

Use this page when you are not sure how to ask Codex for help or when Codex starts to feel too broad.

## The Beginner Loop

1. **Ask** for one clear thing.
2. **Inspect** what Codex can see or what it found.
3. **Plan** before bigger changes.
4. **Act** when you are ready for Codex to edit or create something.
5. **Review** what changed before you trust the result.

## Strong Beginner Habits

- Start with the smallest useful task.
- Tell Codex whether you want explanation, planning, or action.
- Use the right folder: broad enough for context, narrow enough for safety.
- Name the files or folders Codex should use when possible.
- Ask Codex what it is about to change before letting it edit.
- Review changes before trusting them.
- Ask Codex to summarize what changed after edits.
- Keep one goal per prompt while you are learning.
- Use Codex as a tutor when confused, not only as a doer.
- Stop and ask for a recap when you lose your place.

## Helpful Prompts

```text
Before changing anything, explain what you found and what you recommend.
```

```text
Plan this first. Tell me what you would change, which files you would touch, and why.
```

```text
Make the smallest useful change, then summarize what changed so I can review it.
```

```text
I am lost. Summarize where I am in the course and tell me the next sensible step.
```

## When To Slow Down

Slow down and ask for a plan first when:

- the task touches more than one file
- the output will be used by other people
- you do not understand what Codex is about to change
- Codex gives you a very broad answer
- you feel like you are just clicking along without understanding

Slowing down is not a failure. It is one of the main ways experienced Codex users stay in control.
