# Codex Course Cheat Sheet

## Core Agreements

1. **Use AI to ask AI**  
   If you are stuck, ask Codex.

2. **Progress over perfection**  
   A useful first draft is enough to keep moving.

3. **Speed bumps are part of learning**  
   Confusing output is normal when learning Codex.

4. **Review before you trust**  
   Codex can read and edit files here. You still decide what to accept.

## When To Use Codex

| Need | What Codex Helps With |
|------|------------------------|
| Understand a file or folder | Summaries, explanations, walkthroughs |
| Plan work before acting | Breaking tasks into steps |
| Improve a document or workflow | Rewrites, structure, cleanup |
| Build something reusable | Skills and local prototypes |
| Get unstuck | Tutoring and troubleshooting |

## Helpful Prompts

- "Show me the full course outline."
- "What should I do next?"
- "Take me to Module 3."
- "Take me to Lesson 4.02."
- "Summarize where I am and what to do next."
- "I want to stop for now. Remind me where to restart later."

## Course Navigation Prompts

- "Show me the full course outline." -> see what is in the course
- "Take me to the next lesson." -> start or resume the next lesson
- "Skip to Lesson 3.02." -> jump to a lesson
- "Help me navigate the course." -> get help with navigation
- "Summarize where I am before I stop." -> wrap up for now and get a restart point

If you are confused, ask a question. You do not have to choose between getting help and staying in the course flow.

Avoid slash commands in this course. In Codex, slash-prefixed inputs may be interpreted as app commands instead of course navigation.

## Key Concepts

**Workspace** — the folder Codex can currently inspect and work inside.

**Skill** — a reusable set of instructions and resources that helps Codex do one kind of job consistently.

**Automation** — a recurring Codex workflow that runs on a schedule.

**Prototype** — a quick local app, dashboard, or site built to test a useful idea.
